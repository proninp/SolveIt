# [Кэш LRU](https://solvit.space/coding/3219)

Спроектируйте структуру данных, которая реализует ограничения [Least Recently Used (LRU) кэша](https://ru.wikipedia.org/wiki/%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC%D1%8B_%D0%BA%D1%8D%D1%88%D0%B8%D1%80%D0%BE%D0%B2%D0%B0%D0%BD%D0%B8%D1%8F#Least_recently_used_(%D0%92%D1%8B%D1%82%D0%B5%D1%81%D0%BD%D0%B5%D0%BD%D0%B8%D0%B5_%D0%B4%D0%B0%D0%B2%D0%BD%D0%BE_%D0%BD%D0%B5%D0%B8%D1%81%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D1%83%D0%B5%D0%BC%D1%8B%D1%85)).

Реализуйте класс `LRUCache`:

`LRUCache(int capacity)` инициализирует LRU-кэш с положительным размером `capacity` (вместимость).
`int get(int key)` возвращает значение по ключу `key`, если ключ существует в кэше. В противном случае возвращает `-1`.
`void put(int key, int value)` обновляет значение ключа `key`, если он уже есть в кэше.
В противном случае добавляет пару ключ-значение в кэш.
Если после этой операции количество ключей превысит `capacity`, необходимо вытеснить (удалить) ключ,
который использовался давнее всех остальных.


**Пример 1:**

Ввод:
`["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]`
`[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]`

Вывод:
`[null, null, null, 1, null, -1, null, -1, 3, 4]`

**Объяснение:**
```
LRUCache lRUCache = new LRUCache(2);

lRUCache.put(1, 1); // кэш: {1=1}
lRUCache.put(2, 2); // кэш: {1=1, 2=2}
lRUCache.get(1);    // возвращает 1
lRUCache.put(3, 3); // LRU-ключ был 2, он вытесняется. Ключ 3 добавляется, кэш: {1=1, 3=3}
lRUCache.get(2);    // возвращает -1 (не найдено)
lRUCache.put(4, 4); // LRU-ключ был 1, он вытесняется, кэш: {4=4, 3=3}
lRUCache.get(1);    // возвращает -1 (не найдено)
lRUCache.get(3);    // возвращает 3
lRUCache.get(4);    // возвращает 4
```

**Ограничения:**

- `1 <= capacity <= 3000`
- `0 <= key <= 104`
- `0 <= value <= 105`


Будет выполнено не более `2 * 10^5` вызовов методов `get` и `put`.
